IO

发表于 2019-01-22

1.java.io包下
File类：java程序中，此类的一个对象就对应着硬盘中的一个文件或网络中的一个资源。

1.File既可以表示一个文件，也可以表示一个文件目录
2.File类的对象是与平台无关的
3.File类针对于文件或文件目录，只能进行新建、删除、重命名、上层目录等，如果涉及到访问文件的内容，File是无能为力的，只能使用IO流下提供的输入输出流实现。
>4.常把File类的对象作为形参传递给相应的输入输出流的构造器中

2.IO流的结构
见图片

3.IO流的划分
1)按照流的流向不同：输入流 输出流（站位于程序的角度）
2)按照流中的数据单位的不同：字节流 字符流（纯文本文件txt）
3)按照流的角色的不同：节点流 处理流 （直接作用于文件上是节点流（4个），其余都是处理流）

4.重点
抽象基类： 节点流（文件流） 缓冲流(处理流的一种)
InputStream FileInputStream(处理字节) BufferedInputStream
OutputStream FileOutputStream(处理字节) BufferedOutputStream
Reader FileReader(处理字符) BufferedReader
Writer FileWriter(处理字符) BufferedWriter

FileInputStream(int read(byte[] b))
FileOutputStream(void write(b,0,len))
FileReader(int read(char[] c))
FileWriter(void write(c,0,len))

BufferedInputStream(int read(byte[] b))
BufferedOutputStream(void write(b,0,len)) flush()
BufferedReader(int read(char[] c))或(String readLine()) readline()
BufferedWriter(void write(c,0,len))或void write(String str) flush()

注：

1.从硬盘中读入一个文件，要求此文件一定要存在，若不存在则报FileNotFoundException的异常
2.从程序中输出一个文件到硬盘，文件可以不存在。若不存在，则创建一个实现输出，若存在，则将原存在文件覆盖
3.真正开发时，使用缓冲流代替节点流
4.最后要关闭相应的流，先关输出流再关输入流（finally)

5.其他的流
1)转换流：实现字节流与字符流之间的转换
InputStreamReader：输入时，实现字节流到字符流的转换，提高操作效率（数据是文本文件） —->解码：从字节数组到字符串
OutputStreamWriter：输出时，实现字符流到字节流的转换。
—->编码：字符串到字节数组

2)标准输入输出流
System.in：从键盘输入数据
System.out：从显示器输出数据

3)打印流

4)数据流

5)对象流

6)随机存取文件流

枚举类和注解

发表于 2019-01-17

一、枚举类
1.如何自定义枚举类。枚举类：类的对象时有限个的，确定的。
(1)私有化类的构造器,保证不能够在类的外部创建对象
(2)在类的内部创建枚举类的实例。声明为:public static final
(3)若类有属性，属性声明为private final。此属性在构造器中赋值

2.使用enum关键字定义枚举类
>2.1常用的方法：values() valueOf(String name)
>2.2枚举类如何实现接口：
①让类实现子接口，类的对象共享同一套接口的抽象方法的实现
②让类的每一个对象都去实现接口的抽象方法，进而通过类的对象调用被 重写的抽象方法时，执行效果不同
说明：
必须将枚举对象放在最上面，可跟抽象方法，注意中间用逗号隔开。

二、注解
1.JDK提供的常用的三个注解：
@Override: 限定重写父类方法, 该注释只能用于方法
@Deprecated: 用于表示某个程序元素(类, 方法等)已过时
@SuppressWarnings: 抑制编译器警告

2.如何自定义注解
以SuppressWarnings为例进行创建即可

3.元注解
对已有的注解进行注解
Retention:(指明生命周期）SOURCE CLASS RUNTIME
Target:(能修饰什么） TYPE
Documented:javadoc
Inherited:继承性

Generic

发表于 2019-01-17

1.泛型在集合中的使用
1>在集合中不使用泛型
2>在集合中使用泛型
具体例子见TestGeneric

2.自定义泛型类、泛型接口、泛型方法
1>自定义泛型DAO
实例化和继承时可以具体化泛型类型
说明：
①.对象实例化时不指定泛型，默认为：Object。
②.泛型不同的引用不能相互赋值。
③.加入集合中的对象类型必须与指定的泛型类型一致。
④.静态方法中不能使用类的泛型。
⑤.如果泛型类是一个接口或抽象类，则不可创建泛型类的对象。
⑥.不能在catch中使用泛型
⑦.从泛型类派生子类，泛型类型需具体化
2>泛型方法
pulic E get(E e){
return e;
}
如果泛型方法中的泛型(比如E)跟泛型类中的泛型(比如T)不一样，那么泛型方法处可加static，因为若为T，则加载类的时候，T的类型未知，所以报错，而E，是要在使用时确定类型，所以是可以的。

3.泛型与集成的关系
A类是B类的子类，G是带泛型声明的类或接口。则G不是G的子类

4.通配符：？
A类是B类的子类，G是带泛型声明的类或接口。则G<?>是GG的父类
①以List<?>为例，能读取其中的数据，因为无论存储的是什么类型的元素，其一定是 Object类或其子类的。
②然而不可以向其写入数据，因为没有指明可以存放到其中的类型，Null除外

5.? extends A: A及A的子类
? super A: A及A的父类

Collection

发表于 2019-01-16

一、对象的存储
①数组（基本数据类型 & 引用数据类型） ②集合（引用数据类型）
数组弊端：①一旦创建，其长度不可变。②实际存储量不可直接知
二、集合框架

—–Collection接口
—-Set：元素无序，不可重复的集合（类似集合）
—-HashSet（主要的实现类）
—-LinkedHashSet（是HashSet一个子类，遍历时为“顺序”）
—-TreeSet(可以按照添加进元素的指定属性进行排序）
—-List:元素有序，可重复的集合（“动态”数组）
—-ArrayList（主要的实现类）
—-LinkedList(链表实现，用于频繁的插入删除操作)
—-Vector(古老的实现类、线程安全的)
—–Map接口：具有映射关系“key-value对”的集合（类似函数）
—-HashMap（主要实现类）
—-LinkedHashMap（HashMap子类，遍历时为“顺序”）
—-TreeMap（需要按照key所在类的指定属性进行排序，要求key是同 类的)
—-Hashtable:古老实现类，线程安全，不可添加Null键
—-子类:Properties：用来处理属性文件
—–Iterator接口：用来遍历集合Collection元素，Map有自己遍历的方法

Collections工具类，操作Collection及Map的工具类，大部分为static的方法。

说明：
1>.Collection方法：
①add(Object obj),addAll(Collection c1),size(),clear(),isEmpty()
②remove(Object obj),removeAll(Collection c1),retainAll(Collection c1),equals(Object obj),set(int Index,Object obj),contains(Object obj),containsAll(Collection c1),hashCode()
（该组都与equals有关）
③iterator(),toArray()

2>.List增加的方法：
remove(int index),set(int index,Object obj),get(int index),add(int index, Object obj)

添加进List集合的元素（或对象）所在的类一定要重写equals()方法

3>.Set接口使用的方法基本都是Collection接口定义的
1.添加进Set集合中的元素所在类一定要重写equals()和hashCode()，且要求保持一致
2.无序性，无序性不等于随机性.无序性指的是元素在低层存储的位置是无序的
3.不可重复性：向Set中添加进相同元素（内容）时，后面的不能添加进去(size()是不变的）

4>.TreeSet
1.要求添加进集合的元素是同一类的
2.自然排序 VS 定制排序
①自然排序：要求自定义类实现java.lang.Comparable接口并重写其compareTo(Object obj)的抽象方法在此方法中，指明按照自定义类的哪个属性进行排序
②定制排序：创建实现Comparator接口的实现类的对象，在实现类中重写compare()方法，在compare()方法中指明按照元素哪个属性进行排序，将此Comparator接口的实现类的对象作为形参传递给TreeSet的构造器中

要求重写的compareTo(),compare()方法与equals(),hashCode()方法保持一致

5>.Map接口
常用方法：添加、修改put(Object key,Object value),删除remove(Object key),获取get(Object key),遍历用size(),keySet(),values(),entrySet()
1.key值是不可重复的,相当于Set来存，value可以重复，使用Collection来存。
一个key-value对构成一个entry(Map.Entry),entry用Set来存放
2.TreeMap类似于TreeSet

OOP总结

发表于 2019-01-13

一、面向对象三大主线：

1.类及类的成分
2.面向对象的三大特性
3.其他关键字：this super import package abstract static final interface等

1.类及类的成分

1.1 Java程序关注于类的设计。

类从代码的角度：并列关系
从执行、设计的角度：关联关系，继承关系，聚合关系
Class A{

}

Class B{
A a = new A(); //关联关系
}

1.2 类的成分： 属性 方法 构造器 代码块 内部类

1.2.1 属性:

①变量的分类：成员变量（属性） VS 局部变量（方法的形参、方法内部、代码块内部）
基本数据类型（8种） VS 引用数据类型（数组、类、接口，默认Null）
②属性的声明格式：修饰符 数据类型 变量名 = 初始化值
修饰符：public protected（缺省） private
③对属性赋值的操作：1.默认初始化 2.显示地初始化 3.代码块 4.构造器 ；5.调用方法
（2、3根据前后位置顺序执行）
（前四个是初始化，第5个是对属性的修改）

1.2.2 方法：

①格式： 修饰符（其他关键字static final abstract) 返回值类型 方法名 （形参列表）{//方法体}
②方法的重载（Overload) vs 方法的重写（override）
③方法的参数传递机制：值传递（难点）
1>形参是基本数据类型的：将实参的值传递给形参的基本数据类型的变量
2>形参时引用数据类型的：将实参的引用类型变量的值（对应的堆空间对象实体的首地址）传递给形参的引用类型变量。（所以此处形参和实参指向的是同一对象）

1.2.3 构造器

①作用：1.创建类的对象 2.成员变量的初始化
②构造器也是可以重载的

1.2.4 代码块

①作用：初始化类的成员变量
②分类：静态代码块 VS 非静态代码块

1.2.5 内部类

①分类：成员内部类（静态成员 vs 非静态成员） vs 局部内部类（方法内部声明的类）
掌握三点：1>如何创建成员内部类的对象（如：创建Bird类和Dog类的对象）
2>如何区分调用外部类与内部类的变量（尤其是变量重名时）
3>局部内部类的使用 //见TestInnerClass1

1.3 类的初始化（创建类的对象）

1.3.1 如何创建类的对象

如：Person p = new Person();

1.3.2 内存解析

①栈：局部变量、对象的引用名、数组的引用名
堆：new出来的东西
方法区：（字符串常量池）
静态域：存放类中静态的变量
②如何理解创建的对象在内存中加载的过程

1.3.3 子类对象实例化的全过程

SubClass sc = new SubClass();
创建的过程中会加载父类以及父类的父类直至Object，会加载到所有的构造器

2.面向对象的三大特性

2.1封装性

①通过私有化类的成员变量，通过公共的getter和setter方法来调用和修改
②还可以对类的其他结构进行“封装”
③权限修饰符：public protected 缺省 private

2.2继承性

通过让类A继承另一个类B，就可以获取类B中的结构（属性、方法、构造器等）
A extends B
java中类的继承性是单继承的（一个类只能继承一个父类）

2.3多态性

①体现：方法的重载与重写；子类对象的多态性 Person p = new Student();
②子类对象多态性的使用：虚拟方法调用
③向上转型 向下转型
if(p instanceof Student) //避免出现ClassException的异常
Student s = (Student)p;

3.其他关键字

3.1 this

修饰属性、方法、构造器，表示当前对象或当前正在创建的对象

3.2 super

修饰属性、方法、构造器，显示地调用父类的响应的结构，尤其是子父类有重名的方法和属性时

3.3 static

修饰属性、方法、代码块、内部类，随着类的加载而加载

3.4 final

修饰类、属性、方法，表示最终的

3.5 abstract

修饰类、方法
不能与static final private共用

3.6 interface

接口，（与类并列的结构），无构造器，定义的是功能
类与接口之间通过implements发生关系

5.7 package / import
